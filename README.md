# MyScholar AI

A Retrieval-Augmented Generation (RAG) application that enables users to query academic documents and receive accurate, context-aware answers generated by a Large Language Model.

**Live Demo:** [MyScholar AI on Hugging Face](https://huggingface.co/spaces/AremuTaiwo/Rag-Pipeline)  
**Demo Video:** [Watch on YouTube](https://youtu.be/0uXN2yo_eq8)

---

## Overview

MyScholar AI is designed for students and researchers who need to extract information from academic documents without manually reading through entire files. The system indexes uploaded documents, retrieves the most semantically relevant content based on the user query, and generates precise responses grounded strictly in the document context.

---

## Features

- Automatic ingestion of PDF and DOCX documents
- Text extraction, chunking, and embedding generation
- Semantic vector search using ChromaDB
- Context-aware response generation via Groq LLM
- Display of all indexed documents within the interface
- Clean conversational interface built with Gradio
- FastAPI API for programmatic access and local testing
- Containerised deployment on Hugging Face Spaces via Docker
- Structured logging across all modules for production observability
- Unit tests written with pytest covering core pipeline functions

## Architecture

```
        ▼

        │
        ▼
Text Chunking (LangChain Text Splitters)
        │
        ▼
Embedding Generation (Sentence Transformers)
        │
        ▼
Vector Storage (ChromaDB)
        │
        ▼
User Query
        │
        ▼
Semantic Retrieval of Relevant Chunks
        │
        ▼
Response Generation (Groq — llama-3.1-8b-instant)
        │
        ▼
Answer Displayed in Chat Interface
```

## How It Works

At startup, the application downloads documents from a Hugging Face Dataset, extracts and chunks the text, generates embeddings using Sentence Transformers, and stores them in ChromaDB. When a user submits a question, the query is embedded and compared against stored vectors. The most semantically similar chunks are retrieved and passed alongside the question to the Groq LLM, which generates a response grounded exclusively in the retrieved context.

## How to Use

The application is accessible via the live demo on Hugging Face Spaces. No local installation is required.

1. Open the live demo on Hugging Face Spaces.
2. Expand the Available Documents section to view all indexed files.
3. Type a question in the chat input and press Send or Enter.
4. The application will return an answer based strictly on the content of the indexed documents.

## Local API Testing

The FastAPI backend can be run locally for development and debugging purposes using Postman or any API client.

**Setup**

1. Clone the repository:
```
git clone https://github.com/Aremuoluwatobi/rag-pipeline.git
cd rag-pipeline
```

2. Create and activate a virtual environment:
```
python -m venv env
source env/bin/activate
```
On Windows:
```
env\Scripts\activate
```

3. Install dependencies:
```
pip install -r requirements.txt
```

4. Install development dependencies:
```
pip install -r requirements-dev.txt
```

5. Create a .env file in the root directory and add the following environment variables:
```
GROQ_API_KEY=your_groq_api_key
HF_TOKEN=your_hugging_face_token
```

Note: The GROQ_API_KEY is required for LLM response generation. The HF_TOKEN is required for authenticated access to Hugging Face services and document downloads.

6. Start the API server:
```
uvicorn main:app --reload
```

The API will be available at http://127.0.0.1:8000.

## API Endpoints

| Method | Endpoint  | Description                         |
|--------|-----------|-------------------------------------|
| GET    | /         | Confirm the API is running          |
| GET    | /health   | Health check                        |
| POST   | /ingest   | Trigger document ingestion pipeline |
| POST   | /query    | Submit a question and get an answer |

**Example Request**

POST /query
```json
{
    "question": "What is the NATO Warsaw Pact?"
}
```

**Example Response**

```json
{
    "answer": "The NATO Warsaw Pact refers to the two opposing military alliances during the Cold War..."
}
```
